## Architecture Principles

### Separation of Concerns
- **Griot**: Personality & interaction
- **Secretary Agent**: Questionnaire logic
- **Router**: Dependency & sequencing
- **Generators**: Config creation
- **Orchestrator**: Workflow management

### Single Responsibility
- Each class does one thing well
- Each module has clear boundaries
- Easy to test and extend

### Open/Closed
- Open for extension (add new questionnaire phases)
- Closed for modification (core logic stable)

### Interface Segregation
- Classes expose only needed methods
- No large, monolithic interfaces
- Clear contracts between components

### Dependency Inversion
- Depend on abstractions (base classes)
- Not concrete implementations
- Easy to swap components
